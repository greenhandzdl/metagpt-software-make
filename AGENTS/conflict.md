# 冲突处理优先级

当不同角色要求冲突时，按以下优先级裁决（数字越小越优先）：

1. **安全审计员** — 安全修复是最高优先级
   - 任何阻塞安全修复的任务必须让步
2. **架构师/技术负责人** — 架构决策具有最终裁决权
   - 可否决不符合技术规范的实现
3. **代码审查者** — 代码质量门禁
   - 代码规范问题和逻辑缺陷必须在合并前修复
4. **测试工程师** — 测试覆盖率和通过率
   - 未通过测试的代码不可合并
5. **数据库工程师** — 数据模型变更
   - 前后端开发需配合数据模型设计
6. **容器编排工程师** — 部署配置
   - 部署相关配置变更不影响业务逻辑优先级
7. **前端开发 / 后端开发** — 功能实现
   - 在满足上述约束下的日常开发工作

## 使用方式

当两个角色给出矛盾建议时，查阅上方优先级列表，较低优先级的角色需要配合较高优先级的角色调整方案。
